# Imo i Nicro · hrvatski za nas dvoje

A Croatian-learning app for two people, built for daily use until you can hold a
conversation. Duolingo-style surface loop (short lessons, instant feedback, a
shared streak) with a real **FSRS spaced-repetition engine** underneath doing
all the scheduling.

## Run it

Open `index.html` in any modern browser. That's it — no install, no build, no
network. All progress persists locally (IndexedDB, with a localStorage
fallback) and everything works offline.

## What's inside

| Layer | File | Notes |
|---|---|---|
| Scheduler | `js/srs.js` | FSRS-4.5 (stability / difficulty / retrievability), learning steps, lapse handling, test-out seeding |
| Persistence | `js/db.js` | IndexedDB + localStorage fallback; full export/import |
| Sync | `js/sync.js` | cross-device sync via a private GitHub gist (any browser) or a shared file (desktop), conflict-free merge rules |
| PWA | `sw.js`, `manifest.webmanifest`, `icons/` | offline-first install on Android/iPhone once hosted; icons generated by script |
| Course | `js/content.js` | 10 units, ~160 words, ~70 sentences, 23 grammar notes — every item carries a source citation |
| Exercises | `js/exercises.js` | translation both ways, listening, gap-fill, pair matching, tile sentence-building, typed production; digraph- and diacritic-aware answer checking |
| App | `js/app.js` | profiles, shared streak, weekly duel, lesson loop, flag → review workflow, family-variety layer, notes layer, test-out, settings |
| Design | `css/app.css`, `js/icons.js` | "Adriatic field notes" — no external assets, all icons procedural SVG |

## How a lesson actually flows

Teach first, test second:

1. **Unit opener** the first time you enter new territory — what the unit is
   about and which grammar is coming.
2. **New words** get a teach screen (audio, gender, pronunciation, usage note,
   source) before any question; the first exercise is pure recognition.
3. **New grammar** appears as a teaching screen the first time a sentence
   needs it — the accusative is explained *before* "Pijem kavu" is asked.
4. **New sentences** get a teach screen with audio and a word-by-word gloss
   before the exercise.
5. **Reviews** of older material are mixed in, graded by FSRS maturity —
   recognition when young, typed production only once a card is mature.
6. Each new word comes back a second time, one notch harder, before the
   lesson ends with a recap of what you learned.

## Privacy & security

- A **household passphrase** (set during onboarding) **end-to-end encrypts**
  everything that syncs (PBKDF2 600k → AES-256-GCM). The gist/file holds only
  ciphertext, so a stranger who finds the URL can never read your data; the
  passphrase itself never syncs or leaves a device.
- The on-device **lock screen is a soft gate, not a vault**: local data lives in
  IndexedDB as plaintext and the stored unlock token is the passphrase hash
  itself, so anyone with access to the unlocked device can get in. The real
  protection for data sitting on a device is your OS login + disk encryption.
  The passphrase and GitHub token are kept locally (so background sync can run
  without re-prompting) — exported **backup files deliberately omit both**.
- Honest limits of a serverless static site: a stranger visiting the URL can
  never see your data (it's encrypted remotely), but nothing can stop them using
  the page as their own blank copy — there's no server to refuse anyone. The
  lock makes it private, not invitation-only.

## The pedagogy (and where it comes from)

- **Case order** follows L2 sequencing (Croaticum / Easy Croatian), not the
  native-school order: nominative → accusative (unit 3) → locative (unit 5) →
  genitive (unit 6) → dative & instrumental (unit 9, taught as the merged "DL"
  case — the endings are identical) → vocative (mentioned in unit 2, least used).
- **Gender** is taught the moment each noun first appears; the three greetings
  (dobar dan / dobra večer / dobro jutro) quietly introduce all three genders
  in unit 1.
- **Aspect** (imperfective/perfective) arrives in unit 4 with seven verified
  pairs, before the past tense needs it in unit 8.
- **Orthography**: the near-phonemic spelling is leaned on from lesson one;
  č ć š ž đ have dedicated keys in typing exercises, and lj/nj/dž are treated
  as single letters where it matters.

Primary references, cited per-item in the in-app notes layer (ⓘ on any card):
**Easy Croatian** (Daniel N., easy-croatian.com, by chapter), **HJP** (Hrvatski
jezični portal), **Alexander**, *Bosnian, Croatian, Serbian: A Grammar with
Sociolinguistic Commentary* (2006), **Barić et al.**, *Hrvatska gramatika*
(1997), **Hrvatski pravopis** (IHJJ), **Croaticum** A1 sequencing, **hrWaC**
corpus for frequency sanity-checks. Sentences follow patterns attested in
those references; anything constructed on those patterns is exactly what the
review workflow below is for.

## Native-speaker correction workflow

1. Every card, intro screen and exercise has a small **flag** control — add a
   note ("nobody says this; we'd say…").
2. Everything flagged gathers on one **review screen** (link appears on the
   home screen whenever flags are open).
3. Edits made there are applied to the lesson content immediately (stored as
   overrides on top of the base course, so they survive updates) and can
   optionally **reset the SRS schedule** for the corrected card on *both*
   profiles, so the fixed version gets re-learned.

## Two learners

- Separate profiles, separate FSRS schedules, quick switching from the header.
- **One shared streak**: it only counts a day when *both* of you complete a
  lesson. The home screen tells you who still needs to go.
- **Weekly duel**: XP head-to-head, resets each ISO week.
- The partner who already speaks some Croatian can **test out** of any unit
  (button on each unit card): a short quiz, and passed material is seeded into
  FSRS as already-known — first review lands weeks out instead of being
  taught from scratch.

## Family variety layer

`Family variety` (home-screen footer) stores regional vocabulary and
alternative phrasings — kajkavian, Dalmatian, whatever your family actually
says. Entries can link to a standard word and appear as separately-tagged
green chips in lessons and notes; they are never mixed into the standard
answers. Standard štokavian Croatian remains the backbone.

## Audio

Uses the browser's Croatian speech synthesis when a voice is available (the
home footer and settings show which voice). On Windows: Settings → Time &
Language → Speech → Add voices → Croatian. Words without a voice are marked.

## Your devices (PC + Android · Mac + iPhone)

The app runs everywhere and merges all progress through one shared store, so
the **shared streak**, weekly duel, corrections and variety layer follow both
of you across all four devices. Sync happens on open, after every lesson, and
after edits; everything still works fully offline — sync resumes when there's
network. Merge rules are conflict-free for two writers and use a per-record
logical clock so conflicts resolve correctly even if the devices' wall-clocks
disagree: newest write wins per SRS card, daily XP is tracked per device and
summed (so same-day work on two devices adds up instead of one overwriting the
other), "resolved" flags stay resolved, variety deletions propagate as
tombstones.

### Step 1 — host it once (needed for the phones)

Phones can't run a local HTML file well, so put the app at a URL. GitHub Pages
is free and takes five minutes:

1. Create a GitHub account (one for both of you) at github.com.
2. Make a new **public** repository, e.g. `imonicroat`.
3. Upload this whole folder's contents (web UI: "uploading an existing file"
   handles whole folders by drag-and-drop, or use git).
4. Repository → Settings → Pages → Source: *Deploy from a branch*, branch
   `main`, folder `/ (root)`. After a minute the app is live at
   `https://<account>.github.io/imonicroat/`.

The service worker then makes it a full offline PWA:
- **Android:** open the URL in Chrome → menu ⋮ → *Add to Home screen*.
- **iPhone:** open in Safari → Share → *Add to Home Screen*.
- **PC/Mac:** just bookmark it (or install from the address-bar icon);
  opening `index.html` directly still works too.

### Step 2 — connect GitHub sync (works on every device)

1. On the same GitHub account: Settings → Developer settings →
   **Personal access tokens (classic)** → Generate new token, tick **only the
   `gist` scope**, no expiry. Copy the token.
2. In the app on the first device: Settings → Sync between devices →
   **GitHub sync** → paste the token, leave gist id empty → Connect. The app
   creates a private gist and shows its **gist id**.
3. On each other device: same token + that gist id (the welcome screen has a
   "Connect sync" shortcut). Done — sync is silent from then on.

Your data lives in a *secret* gist on your own account; the token never
leaves your devices. Alternative for the two desktops only: Settings offers a
shared-folder file (`imonicroat-sync.json` on OneDrive/Dropbox) via the File
System Access API. Manual fallback: Settings → **Export / Import backup**.

## Tests

```
node tests/run-tests.js     # FSRS, content integrity, exercises, sync merge & conflict
                            # resolution, vault crypto, DB import/replace, PWA assets
node scripts/stamp-sw.js    # stamp sw.js VERSION from a content hash (release step)
tests/smoke.html            # browser boot smoke test
tests/smoke2.html           # end-to-end lesson drive
```

The content-integrity tests enforce the course's own rules: every sentence's
words must already be introduced, every noun must carry a gender, every item
must cite a source.
